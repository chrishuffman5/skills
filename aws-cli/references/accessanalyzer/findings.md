# Findings

### 2.1 `list-findings`

Lists findings generated by the specified analyzer. Only for **external access analyzers**. Use `list-findings-v2` for all analyzer types. **Paginated operation.**

**Synopsis:**
```bash
aws accessanalyzer list-findings \
    --analyzer-arn <value> \
    [--filter <value>] \
    [--sort <value>] \
    [--starting-token <value>] \
    [--page-size <value>] \
    [--max-items <value>] \
    [--cli-input-json | --cli-input-yaml] \
    [--generate-cli-skeleton <value>]
```

**Parameters:**

| Parameter | Required | Type | Default | Description |
|-----------|----------|------|---------|-------------|
| `--analyzer-arn` | **Yes** | string | -- | The ARN of the analyzer |
| `--filter` | No | map | None | Filter criteria with `eq`, `neq`, `contains`, `exists` operators |
| `--sort` | No | structure | None | Sort order: `{"attributeName": "string", "orderBy": "ASC\|DESC"}` |
| `--starting-token` | No | string | None | Pagination token from previous response |
| `--page-size` | No | integer | None | Items per API call |
| `--max-items` | No | integer | None | Total items to return |

**Filter Structure:**
```json
{
    "string": {
        "eq": ["string"],
        "neq": ["string"],
        "contains": ["string"],
        "exists": true|false
    }
}
```

**Output Schema:**
```json
{
    "findings": [
        {
            "id": "string",
            "principal": {"string": "string"},
            "action": ["string"],
            "resource": "string",
            "isPublic": "boolean",
            "resourceType": "string",
            "condition": {"string": "string"},
            "createdAt": "timestamp",
            "analyzedAt": "timestamp",
            "updatedAt": "timestamp",
            "status": "ACTIVE|ARCHIVED|RESOLVED",
            "resourceOwnerAccount": "string",
            "error": "string",
            "sources": [
                {
                    "type": "POLICY|BUCKET_ACL|S3_ACCESS_POINT|S3_ACCESS_POINT_ACCOUNT",
                    "detail": {
                        "accessPointArn": "string",
                        "accessPointAccount": "string"
                    }
                }
            ],
            "resourceControlPolicyRestriction": "APPLICABLE|FAILED_TO_EVALUATE_RCP|NOT_APPLICABLE|APPLIED"
        }
    ],
    "nextToken": "string"
}
```

### 2.6 `update-findings`

Updates the status of specified findings (archive or activate).

**Synopsis:**
```bash
aws accessanalyzer update-findings \
    --analyzer-arn <value> \
    --status <value> \
    [--ids <value>] \
    [--resource-arn <value>] \
    [--client-token <value>] \
    [--cli-input-json | --cli-input-yaml] \
    [--generate-cli-skeleton <value>]
```

**Parameters:**

| Parameter | Required | Type | Default | Description |
|-----------|----------|------|---------|-------------|
| `--analyzer-arn` | **Yes** | string | -- | The ARN of the analyzer |
| `--status` | **Yes** | string | -- | New status: `ACTIVE`, `ARCHIVED` |
| `--ids` | No | list(string) | None | List of finding IDs to update |
| `--resource-arn` | No | string | None | ARN of the resource identified in the finding |
| `--client-token` | No | string | None | Client token for idempotency |

**Output:** None
